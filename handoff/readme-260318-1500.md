- Created watchdog-fix.html with full root cause analysis and fix documentation

## Root Cause
watchdog.py had NO singleton protection. Two instances (one from Task Scheduler vbs, one from another path using system Python) both monitored port 8001 and both called restart() simultaneously. This caused two uvicorn processes to attempt binding port 8001, one crashed, keepalive/watchdog detected "server down", triggered another restart, creating an infinite loop. The terminal flashing was caused by two concurrent netstat + taskkill subprocess calls racing each other.

## Remaining
- User must manually kill all existing watchdog.py processes and restart once via watchdog_silent.vbs
- Long-term: Consider removing one of the two monitoring systems (keepalive.py OR watchdog.py) since both do the same job
