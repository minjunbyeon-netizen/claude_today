POST /api/carry-over 에서 태스크를 복사할 때 log_activity()를 호출하지 않아 user_activity 테이블이 비어 있었음.
수정: carry_over() 루프 내에 log_activity(conn, "task_carry_over", {...}) 추가. 이제 이월 태스크 생성 시 자동 기록됨.
수정 완료 CC 버튼 0x80070002 오류 — wt.exe 방식 제거
wt.exe new-window -d ... -- cmd /k claude.exe 방식이 wt.exe 경로 문제로 실패.
수정: subprocess.Popen(['cmd', '/k', 'title CC:{proj} & claude'], cwd=path, creationflags=CREATE_NEW_CONSOLE)
이미 열린 창 감지: PowerShell로 MainWindowTitle -like 'CC:{proj}*' 탐색 후 앞으로 팝업.
미연결 checkins · weekly_goals 테이블 데이터 없음
테이블 스키마는 존재하지만 UI에서 데이터를 입력/조회하는 화면이 연결되지 않은 상태.
현재 서비스에 영향 없음. 추후 기능 구현 시 활성화 예정.
확인 필요 로컬 DB ↔ 원격 서버 동기화 없음
remote_url: http://137.184.82.157/daily-focus 에 별도 DB 인스턴스가 운영 중.
로컬과 원격은 각자 독립 DB — 데이터가 분리됨.
해결 방안: 한 곳만 사용하거나, API 기반 단방향 sync 구현.
