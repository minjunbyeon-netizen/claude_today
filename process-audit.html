발견된 Gap 및 조치

GAP 1 — PM2 부팅 자동 시작 미등록 CRITICAL

재부팅/로그아웃/Windows 업데이트 시 PM2가 자동으로 켜지지 않음 (시나리오 1,2,3,4). pm2 startup이 Windows에서 "Init system not found" 오류 발생. pm2-windows-startup npm 패키지 또는 Task Scheduler 등록이 필요함.

npm install -g pm2-windows-startup
pm2-windows-startup install
pm2 save # 현재 프로세스 목록 저장

GAP 2 — max_restarts 10회 초과 시 알림 없음 HIGH

시나리오 8,9,10,13에서 PM2 errored 상태 진입 후 사용자에게 알림이 없음. zombie-watchdog(runner.py)이 PM2 상태를 감시하므로 연동 가능. 또는 max_restarts를 높이고 min_uptime을 설정해 정상 기동 후 카운터 리셋.

// ecosystem.config.js 수정 권장
max_restarts: 50, // 10 → 50
min_uptime: "10s", // 10초 이상 살면 정상 기동으로 카운터 리셋
