근본 원인 분석
3개 프로세스의 관계도
1
keepalive.py 2개 — 두 번 실행됐으나 Global\DailyFocusKeepalive 뮤텍스 있음. 두 번째는 즉시 종료. 자체 방어됨
2
watchdog.py 2개 — 뮤텍스 없음. 레지스트리 Task (venv python) + 다른 경로 (system python) 둘 다 살아있음. 60초마다 포트 체크 → 둘 다 동시에 restart() 호출
3
uvicorn 2개 동시 바인딩 — 두 watchdog가 각각 uvicorn app:app --port 8001 실행. 하나만 성공, 하나는 크래시 → keepalive/watchdog가 "죽었다" 감지 → 다시 restart → 무한 루프
4
터미널 창 깜빡임 — 두 watchdog가 레이스 컨디션으로 동시에 netstat + taskkill 실행. CREATE_NO_WINDOW가 있어도 두 프로세스 타이밍 충돌 시 창 노출

watchdog.py 변경 내용 (3가지)
FIX 1
Windows 뮤텍스 싱글턴
두 번째 watchdog.py 실행 시 즉시 sys.exit(0)
레지스트리 Task + 수동 실행 어디서 시작해도 하나만 생존
+ def acquire_singleton() -> object | None:
+     handle = ctypes.windll.kernel32\
+         .CreateMutexW(None, False,
+         "Global\\DailyFocusWatchdog")
+     if GetLastError() == 183:  # EXISTS
+         return None
+     return handle
FIX 2
restart() 내부 Lock
같은 프로세스 내 동시 restart() 호출 차단
non-blocking acquire 실패 시 중복 호출 무시
+ _restart_lock = threading.Lock()

  def restart() -> None:
+     if not _restart_lock.acquire(
+             blocking=False):
+         log("restart in progress")
+         return
      try:
          ...
+     finally:
+         _restart_lock.release()
FIX 3
Grace Period 강화
시작 후 10초 대기 → 포트 첫 체크
restart 후 15초 대기 (기존 5초) → 오탐 방지
RESTART_GRACE = 15  # was 5

  mutex = acquire_singleton()
  if mutex is None:
      sys.exit(0)      # NEW: exit if dup

+ time.sleep(10)  # startup grace

  while True:
      if not is_alive(PORT):
          restart()
          time.sleep(RESTART_GRACE)  # 15s
FIX 4
venv Python 안전 폴백
venv가 없으면 sys.executable로 폴백
시스템 Python / venv 경로 불일치 오류 방지
  venv_py = os.path.join(ROOT,
      ".venv", "Scripts", "python.exe")
+ PYTHON = (venv_py
+     if os.path.isfile(venv_py)
+     else sys.executable)  # fallback

  # was: PYTHON = venv_py (no check)
